Interface which gives access to the CMS: site elements, structure and content. Access can be get from any place of the site by calling Moto.getInstance().

animationIsPlayingproperty
animationIsPlaying:Boolean  [read-write]

Global flag which indicates that the animation is playing.

Often used to prevent animation collisions on page change. At the start of animation this flag is set to true, at the end - to false.

Moto.getInstance().animationIsPlaying = true;

websiteSoundsEnabledproperty 
websiteSoundsEnabled:Boolean  [read-write]

Enable sound effects playback.

Used to turn off sound effects which are called on timeline in fla files. It's necessary to check Moto.getInstance().websiteSoundsEnabled before playing the sound effect. On site initialization this property can be set to true or to false according to the site settings.

registerLayoutModule()method 
public function registerLayoutModule(layoutID:uint, moduleID:uint, module:IMotoObject):void

Initializes layout module that is already on the stage. You need to pass module ID and module instance to that method.

Registration methods are used when you have some objects that are not created dynamically and are already on the timeline, but need to be controlled by user in the control panel. So, you usually lock these objects and initialize them manually.

Parameters
layoutID:uint — Layout ID
 
moduleID:uint — Module ID
 
module:IMotoObject — Module instance
